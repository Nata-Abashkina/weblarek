# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

#### Интерфейс IProduct
содержит в себе поля и типы данных товара

Поля интерфейса:
`id: string` - уникальный номер товара.
`description: string` - описание товара.
`image: string` - ссылка на изображение товара.
`title: string` - название товара.
`category: string` - категория товара.
`price: number | null` - стоимость товара, может иметь определенную цену или не иметь ее вовсе.

#### Интерфейс IBuyer
содержит в себе поля и типы данных покупателя

Поля интерфейса:
`payment: TPayment` - тип оплаты товара, может быть онлайн или оффлайн.
`email: string` - адрес электронной почты покупателя.
`phone: string` - номер телефона покупателя.
`address: string` - адрес доставки.

#### Интерфейс IOrder extends IBuyer
содержит в себе интерфейс `IBuyer` и дополняет его количеством и массивом товаров.

Поля интерфейса:
`buyer: IBuyer` - интерфейс `IBuyer`.
`total: number` - количество товаров.
`items: string[]` - массив товаров.

#### Интерфейс ICardActions
универсальный интерфейс для передачи обработчиков событий (экшенов) в компоненты карточек.

Поле интерфейса:
`onClick` - функция, которая принимает стандартное событие мыши `MouseEvent` и срабатывает при клике на целевой элемент карточки.

#### Интерфейс ICardFullData extends IProduct
Используется для передачи полных данных в карточку детального просмотра `CardFull`.

Поле интерфейса:
`inBasket?: boolean` - необязательный флаг, который сообщает карточке, находится ли данный товар в корзине в текущий момент, для переключения кнопки.

### Модели данных

#### Класс Catalog
каталог товаров.

Конструктор:
`constructor(products: IProduct[])` - принимает массив товаров.

Поля класса:
`private products: IProduct[]` - хранит массив всех товаров, поле доступно только внутри класса.
`private product: null | IProduct = null` - хранит товар, выбранный для подробного отображения, изначально определено как null, поле доступно только внутри класса.

Методы класса:
`setProducts(products: IProduct[]): void` - сохранение массива товаров полученного в параметрах метода.
`getProducts(): IProduct[]` - получение массива товаров из модели.
`getProductByID(id: string): IProduct | null` - получение одного товара по его `id`.
`setProduct(product: IProduct): void` - сохранение товара для подробного отображения.
`getProduct(): IProduct | null` - получение товара для подробного отображения.

#### Класс Basket
корзина покупателя.

Конструктор класса не принимает параметров.

Поля класса:
`private products: IProduct[] = []` - хранит массив товаров, выбранных покупателем для покупки, изначально пустое, поле доступно только внутри класса.

Методы класса:
`getProducts(): IProduct[]` - получение массива товаров, которые находятся в корзине.
`addProduct(product: IProduct): void` - добавление товара, который был получен в параметре, в массив корзины.
`delProduct(product: IProduct): void` - удаление товара, полученного в параметре из массива корзины.
`delBasket(): void` - очистка корзины.
`getPriceBasket(): number` - получение стоимости всех товаров в корзине.
`getCountProducts(): number` - получение количества товаров в корзине.
`hasProduct(id: string): boolean` - проверка наличия товара в корзине по его id, полученного в параметр метода.

#### Класс Buyer
покупатель.

Конструктор:
`constructor(payment: TPayment, email: string, phone: string, address: string)` - хранит следующие данные: вид оплаты; адреc; телефон; email.

Поля класса:
`private payment: TPayment` - вид оплаты, тип оплаты `TPayment` может принимать значения `online` и `cash`, поле доступно только внутри класса.
`private email: string` - email, поле доступно только внутри класса.
`private phone: string` - телефон, поле доступно только внутри класса.
`private address: string` - адреc, поле доступно только внутри класса.

Методы класса:
`updBuyer(buyer: Partial<IBuyer>): IBuyer` - сохранение данных в модели. Один общий метод, реализована возможность сохранить только одно значение, не удалив при этом значения других полей, которые уже могут храниться в классе.
`getBuyer(): IBuyer` - получение всех данных покупателя.
`delBuyer(): void` - очистка данных покупателя.
`validPayAndEmail(payment: TPayment, email: string): {payment?: string; email?: string}` и `validPhoneAndAddr(phone: string, address: string): {phone?: string, address?: string}`- валидация данных. Поле является валидным, если оно не пустое. Метод валидации предоставляет информацию об ошибке, связанной с проверкой конкретного значения.

### Слой коммуникации

#### Класс ApiClient
клиент для работы с API.

Конструктор:
`constructor(api: IApi)` - принимает реализацию интерфейса `IApi`.

Поля класса:
`private api: IApi` - экземпляр API для выполнения запросов, поле доступно только внутри класса.

Методы класса:
`getProducts(): Promise<IProduct[]>` — получает список товаров.
`postOrder(basket: Basket, buyer: Buyer): Promise<IOrderConfirmation>` — отправляет заказ.

### Слой представления

#### Класс Header
отвечает за отображение шапки и взаимодействие с корзиной. Для передачи данных использует интерфейс `IHeader`, который определяет поле `counter`.

Конструктор:
`constructor(protected events: IEvents, container: HTMLElement)` - принимает корневой DOM-элемент и события `IEvents`.

Поля класса:
`protected counterElement: HTMLElement` - счетчик корзины, защищенное поле.
`protected basketButton: HTMLButtonElement` - кнопка открытия корзины, защищенное поле.

Сеттер класса:
`set counter(value: number)` - отображает счетчик товаров.

#### Класс Gallery
отвечает за отображение списка товаров. Дле передачи использует интерфейс `IGallery`, который определяет массив `items`.

Конструктор:
`constructor(container: HTMLElement)` - принимает корневой DOM-элемент, в который будут вставляться карточки.

Сеттер класса:
`set items(cards: HTMLElement[])` - сеттер очищает старое содержимое и добавляет в контейнер массив товаров.

#### Класс Modal
отвечает за отображение модального окна. Использует интерфейс `IModalData`, определяющий контент, помещаемый в модальное окно.

Конструктор:
`constructor(container: HTMLElement, protected events: IEvents)` - принимает корневой DOM-элемент и события `IEvents`.

Поля класса:
`protected contentElement: HTMLElement` - контент, помещаемый в модальное окно, защищенное поле.
`protected closeButton: HTMLButtonElement` - кнопка закрытия модального окна, защищенное поле.

Методы класса:
`open()` - добавляет класс `modal_active` модальному окну.
`close()` - удаляет класс `modal_active` у модального окна.

Сеттер класса:
`set content(value: HTMLElement)` - очищает старое содержимое модального окна и добавляет новое.

#### Класс CardMain<T>
основной класс карточки товара. Класс является дженериком и принимает в переменной `T` специфичные наборы данных дочерних карточек.

Конструктор класса:
`constructor(container: HTMLElement)` - принимает корневой DOM-элемент.

Поля класса:
`protected titleElement: HTMLElement` - название товара, защищенное поле.
`protected priceElement: HTMLElement` - цена товара, защищенное поле.

Сеттеры и геттеры класса:
`set title(value: string)` — устанавливает текст названия товара в элемент `titleElement`.
`get title()` — возвращает текущее название товара (нужно для альтернативного текста картинок в дочерних классах).
`set price(value: number | null)` — устанавливает цену товара. Если цена отсутствует (`null`), выводит текст «Бесценно».

#### Класс CardGallery
класс карточки товара в общем списке товаров. Использует свой тип данных `TCardGallery` из `IProduct` берет только поля `category` и `image`.

Конструктор класса:
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает корневой DOM-элемент и необязательно действия с карточками в списке.

Поля класса:
`protected categoryElement: HTMLElement` - категория товара, защищенное поле.
`protected imageElement: HTMLImageElement` - изображение товара, защищенное поле.

Сеттеры класса:
`set category(value: string)` - устанавливает текст категории товара и меняет цвет плашки в зависимости от категории.
`set image(value: string)` - устанавливает изображение в карточку товара.

#### Класс CardBasket
класс карточки товара в корзине. 

Конструктор класса:
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает корневой DOM-элемент и необязательно действия с товарами в списке.

Поля класса:
`protected indexElement: HTMLElement` - порядковый номер товара в корзине.
`protected deleteButton: HTMLButtonElement` - кнопка удаления товара из корзины.

Сеттер класса:
`set index(value: number)` - устанавливает порядковый номер товару, добавленному в корзину.

#### Класс CardFull
карточка товара, содержащая полное его описание. 

Конструктор класса:
`constructor(container: HTMLElement, actions?: ICardActions)` - принимает корневой DOM-элемент и необязательно действия с товарами в списке.

Поля класса:
`protected categoryElement: HTMLElement` - категория товара, защищенное поле.
`protected imageElement: HTMLImageElement` - изображение товара, защищенное поле.
`protected descriptionElement: HTMLElement` - описание товара, защищенное поле.
`protected buttonElement: HTMLButtonElement` - кнопка покупки/удаления товара, защищенное поле.

Сеттеры класса:
`set price(value: number | null)` - отображает цену товара и делает кнопку неактивной, если у товара отсутствует цена.
`set inBasket(value: boolean)` - проверяет наличие товара в корзине, товар есть - кнопка меняет текст на «Удалить из корзины».
`set category(value: string)` - устанавливает текст категории товара и меняет цвет плашки в зависимости от категории.
`set image(value: string)` - устанавливает изображение в карточку товара.
`set description(value: string)` - устанавливает описание в карточку товара.

#### Класс BasketView
отвечает за отображение корзины. Использует интерфейс `IBasketView`, определяющий список товаров в корзине и их общую стоимсоть.

Конструктор класса:
`constructor(container: HTMLElement, protected events: IEvents)` - принимает корневой DOM-элемент и события `IEvents`.

Поля класса:
`protected listElement: HTMLElement` - контейнер для списка товаров в корзине, защищенное поле.
`protected totalElement: HTMLElement` - общая стоимость товаров, защищенное поле.
`protected buttonElement: HTMLButtonElement` - кнопка оформления заказа, защищенное поле.

Сеттеры класса:
`set total(value: number)` - отображает общую стоимость товаров в корзине.
`set items(cards: HTMLElement[])` - формирует массив товаров в корзине, если товаров нет появляется текст «Корзина пуста» и кнопка оформления заказа неактивна.

#### Класс Form<T>
основной класс формы заказа. Класс является дженериком и принимает в переменной `T` специфичные наборы данных дочерних форм.

Конструктор класса:
`constructor(container: HTMLFormElement, protected events: IEvents)` - принимает корневой DOM-элемент и события `IEvents`.

Поля класса:
`protected submitButton: HTMLButtonElement` - кнопка оформления заказа, защищенное поле.
`protected errorsElement: HTMLElement` - поле с ошибками валидации, защищенное поле.

Сеттеры класса:
`set valid(value: boolean)` - проверяет форму на валидность, если хоть одно поле пустое - кнопка неактивна.
`set errors(value: string)` - выводит список ошибок.

#### Класс FormAddPayment
форма выбора метода оплаты и введения адреса. Из `IBuyer` использует поля `payment` и `address`.

Конструктор класса:
`constructor(container: HTMLFormElement, protected events: IEvents)` - принимает корневой DOM-элемент и события `IEvents`.

Поля класса:
`protected cardButton: HTMLButtonElement` - кнопка онлайн оплаты, защищенное поле.
`protected cashButton: HTMLButtonElement` - кнопка оплаты при получении, защищенное поле.

Сеттер класса:
`set payment(value: string)` - в зависимости от выбранного способа оплаты добавляет класс `button_alt-active` выбранной кнопке.

#### Класс FormEmTel
форма ввода адреса электронной почты и номера телефона. Из `IBuyer` использует поля `email` и `phone`.

Конструктор класса:
`constructor(container: HTMLFormElement, protected events: IEvents)` - принимает корневой DOM-элемент и события `IEvents`.

### Слой управления

#### Класс Presenter
презентер приложения, связывает слой данных и слой отображения.

Конструктор класса:
`constructor(private apiClient: ApiClient, private events: IEvents, private gallery: Gallery, private templateSelector: string = '#card-catalog', private cdnUrl: string = '', private basketModel: Basket, private productsModel: Catalog, private buyerModel: Buyer, private modal: Modal)` - принимает `Api`, брокер событий, компонент сетки, настройки шаблонов, адрес удаленного сервера, модели корзины, каталога товаров и покупателя, а также модальное окно.

Поля класса:
`private orderForm?: FormAddPayment` - форма выбора метода оплаты и ввода адреса,на момент создания класса формы нет, приватное поле.
`private contactsForm?: FormEmTel` - форма для ввода адреса электронной почты и номера телефона, на момент создания класса формы нет, приватное поле.

Методы класса:
`async init(): Promise<void>` - асинхронный метод, запрашивает массив товаров из `Api`, превращает их в готовые элементы карточек и передает в компонент `Gallery`.
`handleCardOpen(item: IProduct): void` - вызывает событие клика по карточке, чтобы передать выбранный товар для открытия в модальном окне.

### Точка входа приложения
`main.ts` является главным скриптом приложения, который запускается первым при загрузке страницы. Он выполняет роль конфигуратора и «клея» — создаёт все необходимые объекты и связывает слои модели данных и представлений между собой.

`main()` - главная функция, асинхронная.

Шаги функции:
- Создаются экземпляры брокера событий `EventEmitter` и клиента для работы с сетью `ApiClient`.
- Создаются центральные хранилища приложения — каталог товаров `Catalog`, корзина `Basket` и модель данных покупателя `Buyer`.
- Находятся базовые HTML-контейнеры на странице и на их основе создаются компоненты шапки `Header`, витрины `Gallery` и универсального модального окна `Modal`.
- Настраивается прямая реактивная подписка шапки на изменения в корзине `basket:change`, чтобы счётчик товаров всегда оставался актуальным.
- Создаётся экземпляр единственного класса `Presenter`, в который передаются абсолютно все созданные модели, компоненты и настройки. Приложение стартует через асинхронный метод `presenter.init()`.
